Задача 2

1)высоконагруженная база данных MySql, критичная к отказу — нужен в любом случае кластер, а выбор между физическим и виртуальным зависит от требуемых мощностей, по моему мнению лучше использовать два физических сервера, на которых будет развернута система виртуализации и виртуальные машины с БД

2)различные web-приложения — это простые паралельные приложения/службы, тут подойдет лучше всего контейнеризация/виртуализация уровня ОС

3)Windows-системы для использования бухгалтерским отделом — если имеется в виду сервер ПО по типу сервер 1С, то лучше использовать физический сервер. Если под VDI то паравиртуализация.

4)системы, выполняющие высокопроизводительные расчёты на GPU — физический сервер, для более удобного использования ресурсов

Задача 3

1)100 виртуальных машин на базе Linux и Windows, общие задачи, нет особых требований. Преимущественно Windows based-инфраструктура, требуется реализация программных балансировщиков нагрузки, репликации данных и автоматизированного механизма создания резервных копий

Использовать физические сервера, на которых будут развернуты ВМ, которые будут взаимодействовать между собой как балансировщики и репликаторы

2)Требуется наиболее производительное бесплатное open source-решение для виртуализации небольшой (20-30 серверов) инфраструктуры на базе Linux и Windows виртуальных машин.

В данном случае лучше всего подойдет Proxmox

3)Необходимо бесплатное, максимально совместимое и производительное решение для виртуализации Windows-инфраструктуры.

Подходит хорошо как и hyper-v, так и proxmox

4) Необходимо рабочее окружение для тестирования программного продукта на нескольких дистрибутивах Linux.

Proxmox
